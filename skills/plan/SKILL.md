# /plan — 구현 계획 생성 스킬

## 개요

사용자 요청을 분석하여 독립적이고 실행 가능한 구현 계획을 생성합니다.

## 실행 흐름

```
사용자 요청
    ↓
1. 요구사항 정제 — 모호한 부분 확인 (최대 3개 질문)
    ↓
2. 코드베이스 탐색 — 영향 범위 파악
    ↓
3. 태스크 분해 — 단일 책임 단위로 분해
    ↓
4. 의존관계 분석 — DAG 구성, parallel_group 배정
    ↓
5. 계획서 출력 — plan_v1 YAML 포맷
```

## 사용법

```
/plan 사용자 인증 시스템 구현해줘
```

또는 자연어로:
```
"이 프로젝트에 로그인 기능을 어떻게 추가할지 계획 세워줘"
```

## 계획 품질 기준

1. **검증 가능한 DoD** — 각 태스크의 완료 기준이 테스트/빌드로 검증 가능
2. **단일 책임** — 하나의 태스크는 하나의 기능/수정만 담당
3. **순환 의존 금지** — DAG만 허용
4. **병렬 최적화** — 독립 태스크는 같은 parallel_group으로 묶기
5. **보수적 추정** — 노력도 의심스러우면 high

## 관련 에이전트

- **planner** — 이 스킬을 실행하는 에이전트 (HIGH 티어 모델)
